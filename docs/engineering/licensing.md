# Licensing

License selection is defined for the main public surfaces.

Written public-facing dissemination content should be reusable with attribution.
Source code is open under MIT and requires preserving the copyright and license
notice. Brand assets, photographs and documents require separate handling.

Code, written content, photographs, brand assets, and the Mikeroguez mark use
different licensing rules. Do not assume that one license covers all public
materials.

Before publication, define:

- content license;
- rules for logo and brand assets;
- rights for photographs and third-party materials.

Current public posture:

- `package.json` declares `MIT`.
- Source code is licensed under MIT through the root `LICENSE` file. Reuse must
  preserve the copyright and license notice.
- The public site includes `/license` with reuse rules.
- Written public-facing content on the site is offered under Creative Commons
  Attribution 4.0 International (CC BY 4.0), unless a page or file states
  otherwise.
- Photographs, PDFs, third-party materials and publication records require their
  own rights review.
- The Mikeroguez name is treated as a public identity and brand name used since
  2008 according to the owner, without claiming trademark registration. The
  owner may seek trademark registration in the future.
- Public assets are tracked separately in
  `docs/engineering/public-assets-inventory.md`.
- Unresolved legal, brand and editorial decisions are tracked in
  `docs/engineering/publication-readiness-pending.md`.
