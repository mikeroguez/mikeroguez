# UX Review

## Purpose

Review whether pages are understandable, navigable and useful.

## When To Use

Use before publishing new pages, navigation, layout or interaction changes.

## Files To Read

- `AGENTS.md`
- `docs/ux/`
- affected views, components and router files

## Inputs Needed

User goal, page goal, viewport considerations and target audience.

## Procedure

1. Identify the page objective.
2. Review hierarchy and navigation.
3. Check comprehension and mobile layout.
4. Review calls to action.
5. Identify friction and accessibility issues.

## Criteria

Objective clarity, hierarchy, navigation, comprehension, mobile behavior,
accessibility, calls to action and friction.

## Output Format

List blockers, then improvements, with recommendations.

## Stop For Human Review

Stop if the UX depends on unapproved content or private evidence.
