# Accessibility Review

## Purpose

Review accessibility blockers and improvements.

## When To Use

Use for UI, routing, layout, form or content changes.

## Files To Read

- `AGENTS.md`
- `docs/ux/accessibility.md`
- affected `src/` files
- tests when applicable

## Inputs Needed

Changed UI, expected interactions and validation results.

## Procedure

1. Run available automated checks.
2. Review keyboard navigation.
3. Review focus visibility.
4. Review structure and headings.
5. Review zoom, reduced motion, contrast and labels.
6. Separate blockers from improvements.

## Criteria

Keyboard, focus, structure, zoom, motion, contrast and accessible names.

## Output Format

Blockers first, then improvements, with file or component references.

## Stop For Human Review

Stop if a user cannot navigate, understand or operate essential content.
