# Security

Do not publish secrets, credentials, private documents, private contact
information, data about students, client information, or confidential
institutional material.

For non-sensitive issues, GitHub Issues may be used.

Do not report secrets, vulnerabilities, or private personal information in a
public issue. A private reporting channel still needs to be defined.

If sensitive information is exposed, stop publication work, preserve evidence,
and request human review before making further public changes.
